## 0. 프로젝트 개요

티가든 별(Teegarden's Star, 지구에서 12.5광년, 양자리)의 행성계 전체를 물리적으로 정확하게 시뮬레이션하고, 특히 **티가든 b의 명암 경계(terminator) 지대에 서 있는 "시민" 관점의 하늘**을 렌더링한다.

핵심 산출물 3가지:

1. **System View** — 항성 + 행성 b/c/d의 궤도 시뮬레이션 (케플러 전파, 시간 가속/스크러빙)
2. **Surface View (Terminator Sky)** — 티가든 b 명암 경계 지표에서 본 실시간 하늘: 수평선에 걸린 거대 적색 항성, 행성 c·d의 합(opposition) 이벤트, 배경 별자리(태양 포함), 플레어
3. **시민 시간 시스템** — 인간 생체리듬(circadian rhythm)에 기반한 티가든 b의 "하루/주" 달력 + 시계 UI

## 3. 궤도역학 엔진

### 3.1 방식: 케플러 전파 (Kepler propagation)

3행성 모두 저질량(≤지구급)이고 평균운동 공명에서 벗어나 있어(P 비율 b:c:d ≈ 1 : 2.33 : 5.33), 시뮬레이션 시간 규모(수년)에서 상호 섭동은 시각적으로 무의미하다. **N-body 불필요. 행성별 독립 케플러 궤도**로 충분하며 수치적으로 무한히 안정하다.

각 프레임마다:

```
M = M0 + n·(t − t_epoch)          // 평균근점이각, n = 2π/P
E − e·sinE = M                    // 케플러 방정식 → 뉴턴-랩슨 3~4회 반복
ν = 2·atan2(√(1+e)·sin(E/2), √(1−e)·cos(E/2))   // 진근점이각
r = a·(1 − e·cosE)
위치 = (r·cosν, 0, r·sinν)  // 공면(coplanar) 가정 [가정], y-up 기준 황도면=xz
```

구현 요구:
- `solveKepler(M, e): E` — 뉴턴-랩슨, 허용오차 1e-10, e<0.1이라 4회 내 수렴
- 단위: **AU / day / M☉**, 시간 기준 **JD(율리우스일)** 내부 사용
- 모든 행성 공면 + i=90° 시점 [가정] (RV 관측이라 실제 i 미지 — UI에 "관측상 미확정" 표기)
- 옵션: 미래 확장용으로 위치 계산 함수를 `propagator` 인터페이스로 추상화 (N-body 교체 가능하게)

### 3.2 시간 컨트롤러

- 상태: `simTimeJD`, `timeScale` (1× ~ 10⁶×), `paused`
- UI: 재생/일시정지, 속도 슬라이더(로그), 타임라인 스크러버, "다음 c 충(衝)으로 점프" 등 **이벤트 점프 버튼**
- `requestAnimationFrame` 델타 기반 적분: `simTimeJD += (dtMs/86400000) * timeScale`
- 시간 표시 3중 병기: ① 지구 UTC 환산 ② 티가든 b 시민시각(§5) ③ 궤도 위상(0–360°)

## 4. 조석 고정(tidal locking) & 칭동(libration) 모델

근거: a=0.0259 AU의 지구질량 행성은 조석 고정 시간척도가 항성 나이(>8 Gyr)보다 압도적으로 짧다. 학계 표준 가정도 1:1 동주기 자전이다 (Boukrouche+2025, 직접영상 연구들 모두 tidally locked 가정 채택). e=0.03으로 작아 수성형 3:2 공명보다 1:1이 유력.

구현:
- b의 자전주기 = 공전주기 (4.90634일), 자전축 경사 0° [가정]
- **항성직하점(substellar point)** = 경도 0°로 정의. 명암경계 = 경도 ±90° 대원
- **광학 칭동**: e≠0이므로 자전(등속)과 공전(케플러 부등속)의 차이로 항성이 평균 위치 주위를 **±2e rad ≈ ±3.4°** 진폭, 주기 4.90634일로 진자운동
  - 진근점이각 ν와 평균근점이각 M의 차 = 중심차(equation of center): `ν − M ≈ 2e·sin M` (e≪1)
  - 항성의 행성 기준 경도 오프셋 = `−(ν − M)` 을 그대로 적용하면 됨
- **결과 (핵심 연출)**: 명암경계 관측자에게 항성 중심의 고도가 −3.4° ~ +3.4°를 왕복. 항성 각반지름이 1.23°이므로 **거대한 적색 태양이 약 117.75시간 주기로 완전히 떠올랐다가 완전히 지는** 느린 일출·일몰이 존재한다. 이것이 티가든 b의 유일한 자연적 "낮-밤" 신호다.

## 5. 인간 생체시계 기준 — 티가든 b의 "하루" 설계

### 5.1 과학적 제약 조건

1. **인간 자유진행 일주기(free-running circadian period) ≈ 24.2시간** (Czeisler et al. 1999). 빛 자극(zeitgeber)으로 대략 **23.5–24.6시간** 범위에 동조(entrainment) 가능. 그 밖이면 만성 시차증 유발.
2. 조석 고정 행성에는 고정 지점 기준의 자연 명암주기가 **거의 없다.** 유일한 주기 신호는 §4의 **칭동 일출·일몰(117.75시간 주기)**.
3. **M왜성 빛의 생체시계 문제**: 인간 생체시계 동조는 멜라놉신(~480nm 청색광)이 담당하는데, 3,034K 항성 스펙트럼은 청색 성분이 극도로 빈약하다. 주간면 직사광이라도 멜라노픽 자극은 지구 햇빛의 수% 미만 → **자연광만으로는 어디서든 생체시계 동조 불가. 인공 청색강화 조명이 필수 인프라.** (시뮬레이션의 "도시 조명" 연출 근거)

### 5.2 권장 시간계: "1 궤도 = 1 주(週) = 5 시민일" 체계

```
1 티가든 주(Teegarden week) = 1 공전 = 117.75 시간
1 시민일(civic day)        = 117.75 / 5 = 23.55 시간   ← 인간 동조 범위(23.5h↑) 안
1 시민시(civic hour)        = 시민일/24 = 58.9 지구분
```

선택 근거 (대안 비교):

| 방안 | 시민일 길이 | 장점 | 단점 |
|---|---|---|---|
| 궤도÷4 | 29.44 h | 달력 단순 | 동조 불가능 (수면장애 확정) |
| **궤도÷5 (권장)** | **23.55 h** | **동조 가능 + 달력이 천문 현상과 완전 동기** | 지구 24h보다 27분 짧음 (적응 가능) |
| 궤도÷6 | 19.63 h | — | 동조 불가능 |
| 지구 24h 유지 | 24.00 h | 생리적으로 가장 편함 | 달력이 궤도와 어긋나 칭동 일출이 매주 1.05시간씩 표류 — 천문 이벤트가 시계와 무관해짐 |

**권장안의 미학**: 시민일 5일 = 정확히 1 칭동 주기이므로, "주의 시작(1일차 아침)"을 **칭동 일출**(거대 적색 태양이 수평선 위로 완전히 떠오르는 순간)에 고정할 수 있다. 시민들은 5일마다 한 번 진짜 일출을 본다 — 이것이 안식일/주말 문화의 천문학적 근거가 된다. 나머지 4일의 "아침/밤"은 도시 인공조명 사이클로 구현.

추가로 달력 상수: **1 시민년 후보** = c 충 주기와의 최소공배수 등은 과도설계이므로, 단순히 `궤도 카운트(week #)`와 `시민일(1–5)`만 사용. c 충(8.60일)·d 충(6.04일)은 달력과 비동기적으로 도는 "움직이는 명절"로 표시 (지구의 음력 보름과 유사한 위상 드리프트 — 오히려 풍부한 연출 요소).

### 5.3 시계 UI 사양

- 원형 시계: 23.55h를 24 시민시로 분할, 외곽 링에 칭동 고도(태양 위치) 게이지
- "주간 진행 바": 5칸 (1일차=칭동 일출 ~ 5일차=일몰 후 심야), 현재 칸 하이라이트
- 다음 이벤트 카운트다운: 칭동 일출 / c 충 / d 충 / (랜덤) 플레어 경보

## 6. 명암 경계(terminator) 하늘 시뮬레이션 — 핵심 명세

### 6.1 관측자 정의

- 위치: 티가든 b 표면, 경도 +90°(저녁 터미네이터, 기본값) 또는 −90°(아침), 위도 0–45° 선택 가능
- 카메라: 1인칭, FOV 60–75°, 수평선 기준 고도/방위 조작 (OrbitControls 변형 or 커스텀 look controls)
- 지표 기후 맥락 [관측, Boukrouche+2025 GCM]: 지구형 1bar N₂/O₂ 대기 가정 시 전구 평균 306K, 주간면 최고 339K, 야간면 최저 271–285K, 주야 온도차 ~50K. **터미네이터 지대가 280–300K로 가장 온화** → "시민이 사는 이유"의 과학적 근거. 주간면은 증발 우세(건조), **강수는 야간면에 집중** → 터미네이터 상공으로 야간면 구름이 흘러들어오는 연출 가능.

### 6.2 항성 렌더링 (하늘의 주인공)

| 항목 | 사양 |
|---|---|
| 각지름 | **2.47°** — 보름달 4.6개 너비. 수평선에 걸린 거대 원반 |
| 위치 | 방위 = 항성직하점 방향(서쪽 정의), 고도 = 칭동값 −3.4°~+3.4° (주기 117.75h) + 위도 보정 |
| 기본색 | #FFBA70 (3,034K), 수평선 소광으로 짙은 주황~적색 그라데이션 |
| 디스크 셰이더 | 2차 limb darkening: `I(μ)=1−a(1−μ)−b(1−μ)²`, a=0.9, b=−0.2 [가정] |
| 표면 디테일 | 절차적 쌀알무늬(granulation) 노이즈 + 흑점 2–4개 (자전 97.56일로 매우 느리게 이동 — 사실상 정지로 보임) |
| 채층/홍염 가장자리 | 옅은 적색 글로우 림 (M왜성 Hα 활동 표현) |
| 플레어 이벤트 | 푸아송 발생 (λ=0.026/일). 발생 시 130–600초 동안: 국소 백청색(10,000K) 광점 → 디스크 전체 밝기 수십% 상승 → 하늘 산란색이 일시적으로 푸르게 변함 → 지수 감쇠. **시뮬레이션 최고의 드라마 이벤트** |
| 밝기 | 주간면 직사 기준 시각 조도 ~1,700 lux 상당 [유도] — 노출(톤매핑) 기준값으로 사용. 터미네이터에선 영구 시민박명(civil twilight)급 |

### 6.3 행성 c와 d — 시민들의 "달"

c와 d는 모두 b보다 바깥 궤도 → 지구에서 본 화성처럼 **충(opposition) 때 가장 가깝고 보름 위상**이 된다.

| | c | d |
|---|---|---|
| 충 주기 | **8.60일마다** | 6.04일마다 |
| 충 시 각지름 / 밝기 | **15.2′ (보름달의 절반), V ≈ −6.5** | 5.2′, V ≈ −3.0 (금성급) |
| 구상(quadrature, 90° 이각) 시 | 8.0′, V ≈ −5.0 | 3.7′, V ≈ −2.2 |
| 합(conjunction) 시 | 4.2′, V ≈ −3.7 (단, 항성에 근접해 실관측 곤란) | 2.6′, V ≈ −1.5 |
| 위상 | **항상 91~100% (gibbous~full)** — 외행성이므로 충·합 모두 보름, 구상에서 최소 91%. 초승달·반달 위상은 절대 없음 (최대 위상각 34.7°) | 항상 97~100% (최대 위상각 19.1°) |
| 원반 표면밝기 | ~8.0 mag/arcsec² = **보름달의 약 1/70** — 개기월식 때의 구릿빛 달 같은 질감 | ~9.2 mag/arcsec² |
| 합 시 엄폐 | 공면 가정 시 매 회합주기마다 **2.47° 항성 디스크 뒤로 ~1.4시간 엄폐(occultation)** — 터미네이터에선 수평선의 적색 태양 속으로 사라지는 연출 | ~1.0시간 엄폐 |
| 렌더 | 실제 구체 + 위상 라이팅(광원=항성) + 거리 기반 각크기. 밝기는 등급→HDR 강도 변환, **표면밝기 보존** 필수 (점광원 글레어로 처리하면 안 됨 — 원반임) | 동일 |

**적도 터미네이터 관측자의 가시성 캘린더 (핵심)** [유도]:

궤도면이 관측자의 "항성 수평선점–천정–반항성 수평선점"을 잇는 대원과 일치하므로, c·d는 이 대원을 따라 회합주기당 1회전한다. 결과:

- **회합주기의 정확히 절반만 지평선 위** — c는 4.30일 보이고 4.30일 안 보임, d는 3.02일/3.02일
- 사이클: 합(항성 옆 수평선, 엄폐) → ~2.15일 후 **천정 통과** (구상, V≈−5.0) → ~4.30일 후 **충 = 반대쪽(밤 방향) 수평선** — 즉 최대 크기·최대 밝기의 "보름 c"는 어두운 지평선에 걸린 채 뜨거나 지는 장면으로 나타난다 (천정에서가 아님!)
- 저녁/아침 터미네이터에 따라 진행 방향(천정 경유가 합→충 구간인지 충→합 구간인지)이 뒤집힘
- 충의 발생 고도는 관측자 경도에 따라 다름: 반항성점(자정점) 관측자에겐 천정에서, 터미네이터 관측자에겐 수평선에서 발생

구현 노트: c·d의 겉보기 등급은 매 프레임 `m = m_star_from_b + 2.5·log10(1/(A_g·Φ(α)·(R_p/Δ)²·(a_b/a_p)²))` 로 계산 (Φ=램버트 위상함수, Δ=실시간 b–행성 거리). 본 문서 §1.3 및 위 표의 충/구상/합 3개 지점 값과 일치하는지 단위테스트로 검증할 것.

### 6.4 배경 천구 — "지구의 하늘 + 새 별 하나"

12.5광년 이동은 대부분의 항성 위치를 거의 바꾸지 않는다 → **지구 별자리 그대로 사용 가능** (가까운 별 몇 개만 수 도 이동: 시리우스, 프로키온 등 — 1차 버전에서는 무시 가능 [가정]).

- 데이터: **HYG Database v3** (CSV, ~12만 별) → V ≤ 6.5 필터(~9,000개) → 전처리 스크립트로 JSON/바이너리 변환 → `THREE.Points` + 등급별 크기/밝기 + B−V 색지수→색
- **태양 추가 (필수 연출)**: RA 14h53m, Dec −16°53′ (천칭자리), V=2.75 — "시민들이 고향을 가리키는 별". UI에 라벨 토글
- 은하수: 텍스처 돔 또는 가이아 기반 밀도맵
- 천구 회전: b는 동주기 자전이므로 **별이 4.90634일에 한 바퀴** 도는 매우 느린 일주운동 (항성은 칭동 빼면 고정인데 별들만 흐른다 — 독특한 포인트)

### 6.5 대기 산란 — M왜성 전용 스카이 셰이더

기존 Preetham/Hosek 태양 기준 스카이 모델은 **사용 불가** (입력 스펙트럼이 다름). 커스텀 구현:

1. 입력 광원 = 3,034K 플랑크 분포 (8–16 파장 샘플, 380–780nm)
2. 파장별 Rayleigh(λ⁻⁴) + Mie 단일산란 적분 (analytic single-scattering 또는 사전계산 LUT)
3. 결과 스펙트럼 → CIE XYZ → sRGB 톤매핑
4. 예상 외관: 항성 방위는 짙은 호박색~적색 글로우, 천정 방향은 급격히 어두워져 **낮에도 밝은 별이 보이는 암청회색**. 터미네이터에선 영구적 박명 그라데이션 (한쪽 수평선만 영원히 노을)
5. 플레어 시: 광원 스펙트럼에 10,000K 성분 가산 → 하늘이 수 분간 푸른 기 도는 색으로 변화 (사람들이 "하늘이 깨어났다"고 부를 만한 이벤트)
6. 대기 굴절: 수평선 부근 +0.5°가량 떠올림 효과 (선택적, 칭동 일출 타이밍에 영향)

대기 시나리오는 [가정]이므로 프리셋 3종 토글 제공: ① Earth-like 1bar (기본, Boukrouche+2025 근거) ② 얇은 대기 0.1bar (산란 미약, 하늘 검정) ③ 대기 없음 (우주 노출 — 비교용)

### 6.6 기후·날씨 비주얼 (선택 구현, 2025 GCM 근거)

- 야간면 방향 수평선: 강수 구름대·번개 (GCM: 강수는 야간면 집중)
- 주간면 방향: 건조 지형 톤, 상승 적운(항성직하점 쪽 원경)

- 오로라(선택): 플레어 후 수 시간 동안 야간 하늘 방향에 미약한 커튼 — M왜성 항성풍 상호작용 연출

### 7.3 스케일 전략 (중요 결정사항)

실측 비율은 시각화 불가능 (항성 반경 0.00056 AU vs 궤도 0.026–0.079 AU, 행성 반경은 그보다 10⁻² 작음).

- **System View**: 거리=실척(AU 단위 좌표), **천체 반경만 별도 배율** (항성 ×8, 행성 ×400 등) + UI 토글 "True scale" (실척 보기 — 행성이 점이 되는 것 자체가 교육 포인트)
- **Surface View**: 거리 개념 없음 — 모든 천체를 **각도 좌표(고도/방위) + 겉보기 각크기**로 천구돔(반경 고정)에 투영. 물리값은 전부 각도/등급으로 환산해 사용 (§1.3 표가 검증 기준)
- 카메라 정밀도: System View에서 AU 단위 사용 시 부동소수 문제 없음 (0.03–0.08 범위)

### 7.4 셰이더 목록

1. `starSurface.frag` — 흑체색 + limb darkening + 시간 노이즈(granulation) + 흑점 + 플레어 핫스팟
2. `skyScattering.frag` — §6.5 파장샘플 산란 (uniform: sunDir, sunColorTempK, flareBoost, atmosPreset)
3. `planetPhase.frag` — 램버트 위상 + 야간면 미광 + 대기 림(선택)
4. `starfield.vert/frag` — 등급→pointSize/alpha, B−V→색

### 7.5 성능

- 별 필드: 단일 BufferGeometry Points (9k 정점) — 문제 없음
- Bloom은 selective(레이어 분리)로 항성·플레어·c/d에만
- 산란 셰이더: 모바일 대비 파장 샘플 수 quality 옵션 (4/8/16)
- 목표 60fps @ 1080p 데스크탑, 30fps 모바일

### 7.6 디자인 디렉션 (UI)

- 주제가 곧 무드: **"영원한 박명의 도시"** — UI 배경은 터미네이터 하늘에서 추출한 암청회+호박색 투톤, 포인트 컬러는 항성색 #FFBA70 계열 1개로 절제
- 타이포: 데이터(등급·각도·JD)는 모노스페이스/타뷸러 숫자, 라벨은 산세리프 — 천문 소프트웨어의 계기판 감각
- 시그니처 요소 1개에 집중: **시민 시계(§5.3) 원형 위젯** (칭동 고도 게이지 내장) — 이 프로젝트만의 얼굴
- 헤더·박스 남발 금지, 수치엔 항상 단위와 [관측/유도/가정] 배지. 빈 상태/로딩에도 천문 데이터 출처 한 줄 노출

### 7.7 구현 마일스톤

1. **M1**: 데이터 파일 + 케플러 엔진 + 단위테스트 (§10) + System View 기본 궤도
2. **M2**: 시간 컨트롤러 + 시민 시계 HUD (§5)
3. **M3**: Surface View — 천구돔 + 거대 항성(칭동 포함) + HYG 별 + 태양 마커
4. **M4**: c/d 실시간 위치·위상·등급 + 충/엄폐 이벤트 캘린더
5. **M5**: 대기 산란 셰이더 + 플레어 시스템 + Bloom/톤매핑
6. **M6**: 지형·도시 불빛·날씨 연출 + 설정 패널 + 교육 오버레이

## 8. 명시적 가정 목록 (시뮬레이션 내 "Assumptions" 패널에 표기할 것)

1. 궤도 공면, 관측자 시점 i=90° — RV 관측 한계로 실제 3D 기하 미지
2. 질량 = m·sin i (하한값), 반지름 = 암석질 질량–반지름 관계 추정 (트랜짓 없음 → 실측 불가)
3. b는 1:1 동주기 자전, 자전축 경사 0°
4. c·d 이심률 0 (b만 e=0.03 관측)
5. 대기: 기본 시나리오 = 지구형 1bar N₂/O₂ (Boukrouche+2025 GCM 채택 구성) — 실제 대기 존재 여부는 미관측
6. 행성 알베도 0.3 (기하), 위성 없음, 고리 없음
7. epoch 위상(meanLongitudeAtEpochDeg)은 임의값 — 논문 RV 해의 T_periastron으로 교체 가능하도록 상수화
