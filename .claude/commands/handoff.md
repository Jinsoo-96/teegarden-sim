지금 즉시 새로운 구현 작업을 멈추고, 아래를 순서대로 수행하라.
목적: 이 대화의 컨텍스트가 사라져도 다음 세션이 5분 안에 정확히 이어받게 하는 것.

1. 열려 있는 변경사항 저장 후 `npm test` 실행 — **실패해도 고치려 들지 말고 결과만 기록**
2. git 커밋:
   - 태스크 완료 상태면: `M{n}-{k}: <한 줄 요약>`
   - 미완성이면: `WIP(M{n}-{k}): <어디까지 했는지>`
3. `PROGRESS.md` 갱신:
   - NOW 태스크: 완료면 DONE으로 이동(커밋 해시 포함)하고 NEXT 맨 위를 NOW로 승격, 미완이면 NOW에 유지
   - 필요한 경우 DECISIONS / KNOWN ISSUES에 이번 세션 항목 추가 (각 1줄)
   - **HANDOFF NOTE를 아래 5개 항목으로 완전히 덮어쓰기** (이전 내용에 누적 금지):
     - 마지막 작업 파일/함수:
     - 어디까지 했나: (1–3줄, 코드 인용 금지)
     - 다음 세션 첫 행동: (실행할 명령어/수정할 파일/스펙 § 번호 수준으로 구체적으로)
     - 주의사항/함정: (이번 세션에서 발견한 것만)
     - 테스트 상태: X passed / Y failed (실패 테스트 이름 나열)
4. 그 외의 대화 요약, 별도 메모 파일, 코드 덤프를 만들지 말 것 — PROGRESS.md와 git이 유일한 기억이다.
5. 마지막 출력으로 HANDOFF NOTE 내용을 그대로 한 번 보여주고 종료.
